# 計畫功能
# 用法: /plan [功能描述]

使用計畫模式思考如何實作功能：

## 計畫流程

1. **理解需求**
   - 分析功能描述
   - 確認驗收標準
   - 識別相關的現有程式碼

2. **技術設計**
   - 需要建立哪些新檔案
   - 需要修改哪些現有檔案
   - 資料模型設計
   - UI/UX 設計

3. **實作步驟**
   - 將工作拆分成小步驟
   - 每個步驟的預估複雜度
   - 依賴關係和執行順序

4. **風險評估**
   - 可能遇到的問題
   - 對現有功能的影響
   - 測試策略

## 輸出
詳細的實作計畫，等待用戶確認後再開始實作。

所有輸出使用繁體中文。
