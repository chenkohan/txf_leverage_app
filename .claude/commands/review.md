# 程式碼審查
# 用法: /review [檔案路徑]

對指定檔案（或最近修改的檔案）進行程式碼審查：

## 審查項目

### 1. 程式碼品質
- 是否遵循 Dart/Flutter 最佳實踐
- 命名是否清晰易懂
- 函數是否保持單一職責
- 是否有過長的函數需要拆分

### 2. 效能
- 是否有不必要的重建 (rebuild)
- 是否正確使用 const
- 是否有記憶體洩漏風險

### 3. 安全性
- 是否有硬編碼的敏感資訊
- API Key 是否正確處理

### 4. 可維護性
- 是否有足夠的註解
- 程式碼結構是否清晰
- 是否容易測試

## 輸出格式
- 🔴 必須修復的問題
- 🟡 建議改善的項目
- 🟢 做得好的地方

所有輸出使用繁體中文。
